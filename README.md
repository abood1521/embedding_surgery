# Embedding Surgery â€” Part 1 (Finance domain)

We take the general-domain embedding model **`sentence-transformers/all-MiniLM-L6-v2`** and surgically expand its vocabulary with 15 finance terms it previously shattered into subword fragments (e.g. `securitization` â†’ `sec ##uri ##ti ##zation`). After surgery each term is a single token with its own embedding, and everything the model already knew is untouched.

## The surgery in one paragraph

`tokenizer.add_tokens(...)` adds the 15 terms to the vocabulary, and `model.resize_token_embeddings(...)` grows the embedding matrix from 30,522 to 30,537 rows. **Old tokens** keep their pretrained embeddings exactly (verified bit-for-bit against a pre-surgery snapshot). **New tokens** are initialized as the mean of the embeddings of the subword pieces they used to split into, so each starts in a sensible region of the embedding space instead of random noise. Verification checks both directions: what changed is correct, and what didn't change is untouched.

## Main deliverable

- **`main_notebook.ipynb`** â€” the full surgery, executed with outputs: before/after tokenization, the surgery itself, verification, save + reload from disk, end-to-end embedding of finance sentences, and a push-to-Hub cell.
- **`finance-minilm-l6-v2/`** â€” the surgically-modified model + expanded tokenizer (`save_pretrained` output, loadable with `AutoTokenizer`/`AutoModel`). Not committed to git (~90 MB); it is regenerated by running the notebook and published on the HuggingFace Hub:
  **https://huggingface.co/adammoood/finance-minilm-l6-v2**

## How the 15 tokens were chosen

1. Train a 10k WordPiece vocabulary on a finance corpus â†’ `Finance_vocabulary.txt`
2. Download the base model's vocab: `py download_vocab.py microsoft/MiniLM-L12-H384-uncased vocab_1.txt` (all-MiniLM-L6-v2 uses the same 30,522-token BERT vocab) â†’ `vocab_1.txt/vocab.txt`
3. Diff them: `py compare_vocabs.py vocab_1.txt/vocab.txt Finance_vocabulary.txt -o finance_new_vocab.txt` â†’ 4,398 tokens missing from the base model
4. Hand-pick 15 distinctly financial whole words from that diff â†’ **`domain_tokens.txt`** (securitization, collateralized, counterparty, underwriting, hedging, liquidity, volatility, amortization, refinancing, receivables, covenants, tranche, escrow, annuity, actuarial)
